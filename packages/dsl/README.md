# pdum.dsl

Type-keyed DSL compiler infrastructure: reflection capture, the two-tier
type-keyed cache, the Node/Region IR with rewrite and lowering machinery,
marshaling, the registry and its extension surfaces, the events seam, the
value language (device functions over `is_bits` value types), and the
reference evaluator.

Part of the [pdum_dsl](https://github.com/habemus-papadum/pdum_dsl)
workspace; published in lockstep with its sibling `habemus-papadum-*`
distributions. See `docs/design/200_the-spec.md` for the specification.
