7The BASE DIALECT ships as an ``install(registry)`` function — importing this 

8package calls ``install(DEFAULT)`` (batteries), but a hand-built ``Registry`` 

9can receive the exact same dialect explicitly (test isolation; multi-registry 

10sessions; the step-10 ``extend()`` story needs this seam anyway). Contents: 

11the base-language lowering pack, the "device" role (`@jit`'s default — role 

12vocabularies ship with their owning package, per the ch04 decision), the pipe 

13fusion rule, and the pipeline dispatcher (`value > pipeline` executes through 

14the same two-tier path as a call). Chapters before ch09 registered these by

24def install(registry: Registry) -> Registry: 

25 """Register the base dialect into ``registry`` (idempotent).""" 

26 registry.lower_rules.update(LOWER_RULES) 

27 registry.derived.update(PIPE_BUILDERS) 

28 register_role("device", hint="the base language's neutral composable kernel") 

29 register_composition("pipe", "device", "device", "fuse") 

30 if _comb._DISPATCHER is None: # live check — never clobber a dispatcher installed first
